What this measures

This evaluation uses real property records from New York City's PLUTO dataset — the same kind of raw government data that feeds into real estate platforms, appraisals, and lending decisions. The records arrive with ALL-CAPS addresses, cryptic building codes, missing fields, and data entry errors. Three tasks measure whether AI can clean this data reliably: standardize addresses to a consistent format, decode building classifications into human-readable descriptions, and flag data quality issues that would otherwise go unnoticed.

Why this task matters most

Address standardization and classification are predictable — you can write rules for most cases. Data quality auditing is where AI adds the most value: it catches anomalies a human would miss when scanning thousands of records. The "4951 CHAMBERS STREET" catch alone could prevent a bad address from propagating through lending and appraisal systems.

Fix: use AI for the heavy lifting, then apply deterministic post-processing rules for directionals and street type abbreviations. Building Classification

90.0% accuracy on real data

HUMAN-IN-LOOP

The AI confidently produced wrong descriptions for obscure codes like D2 ("Artists in Residence") — it hallucinated a plausible answer instead of saying "I don't know." For a data product, a confident wrong answer is worse than no answer. Use AI as a first pass, but flag codes below a confidence threshold for human verification. Consider maintaining a deterministic lookup table and only using AI for codes not in the table.

Bottom line

On real government property data, AI accuracy ranges from 90-94% depending on the task. That's strong enough to automate the bulk of data cleaning — but not strong enough to trust blindly. The key finding: the AI's worst failures are the confident ones — hallucinating a plausible building class description, or making a bad address look clean. A production pipeline needs both AI for speed and deterministic rules + human review for safety. This harness tells you exactly where to draw that line, task by task, backed by numbers.
